What is DomainLang? ‚Äã

DomainLang is a domain-specific language (DSL) for expressing Domain-Driven Design architecture models. It provides a clean, readable syntax for defining domains, bounded contexts, context maps, teams, and ubiquitous language‚Äîall in plain text files that live in your repository.

üöß Prerelease

DomainLang is in active development and has not reached v1.0 yet. The language syntax may evolve. See the roadmap for planned features.

Why DomainLang? ‚Äã

The problem ‚Äã

DDD models are often:

  • Locked in proprietary tools ‚Äî hard to version control or review
  • Scattered across documents ‚Äî wikis, Confluence pages, diagrams
  • Quickly outdated ‚Äî disconnected from the codebase they describe
  • Hard to validate ‚Äî no tooling to catch inconsistencies

The solution ‚Äã

DomainLang keeps your DDD models:

  • In the repo ‚Äî version controlled alongside code
  • Reviewable ‚Äî changes visible in pull requests
  • Validated ‚Äî IDE catches issues as you type
  • Programmable ‚Äî query and analyze with the Model Query SDK

Key features ‚Äã

DDD-aligned syntax ‚Äã

Express DDD concepts naturally:

dlang
Domain Sales {
    description: "Revenue generation"
    vision: "Make it easy to buy"
}

bc Orders for Sales as CoreDomain by SalesTeam {
    description: "Order lifecycle"
    
    terminology {
        term Order: "A customer's request to purchase"
    }
}

IDE support ‚Äã

The VS Code extension provides:

  • Syntax highlighting for .dlang files
  • Real-time validation and error messages
  • Code completion for keywords and references
  • Hover information and documentation
  • Go-to-definition navigation

Context maps ‚Äã

Model relationships between bounded contexts:

dlang
ContextMap SalesSystem {
    contains Orders, Billing, Shipping
    
    [OHS] Orders -> [CF] Billing
    [ACL] Shipping <- Orders
}
